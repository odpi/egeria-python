# Security glossary

This file include the commands to create a glossary of security terms.
This glossary supports the development of a security strategy.

It includes the following glossary terms:

* api, 
* security, 
* vulnerability, 
* mitigation
* threat modelling, threat landscape, threat model
* security controls,
* risk assessment

Security Glossary

## Language

## Description

This glossary provides definitions for key terms related to security.  It includes definitions for security concepts, types of threats and attacks, security controls, and security approaches.

# Create Glossary Term

## Display Name

Security

## Glossary Name

Glossary::Security-Glossary

## Summary

Security is a state of being in which a person, organization, location, or system is protected against potential harm.

## Description

The focus of security (such as a person, organization, location, or IT system) is called the **security referent**.  The security referent has a relationship with their environment, sometimes called their **security context**.  This may be beneficial, or hostile (or most likely a mixture of both).   The security referent has **security capabilities** that they can use to protect themselves from the hostile forces in their environment.

The **security strategy** lays out how the referent's security capabilities are used to protect themselves.

## URL

https://en.wikipedia.org/wiki/Security

## Abbreviation

Sec

## Usage

The term is used to both describe the state of being secure, and the activities, defences and tools used to protect against security threats.
